1. 要件定義書 (Functional Requirements Specification)
1.1 システム概要
本システムは、After Effects（AE）上でのキャラクターアニメーション制作を効率化するAI拡張機能である。キャラクターの身体的構造を「コンポーネント」としてSupabaseに定義・保存し、その制約情報をLLM（OpenAI/Claude等）に動的に提供することで、特定のキャラクターに最適化されたExtendScript（JSX）を生成・実行する。

1.2 機能要件（詳細）
F-1: キャラクター・スキーマ・キャプチャ
レイヤー構造の解析: ユーザーが選択した複数のレイヤーを解析し、親子関係（Parenting）、名前、種類（シェイプ、ヌル、フッテージ）を特定する。

物理パラメータの自動抽出: 各レイヤーの現在の座標（Position）、アンカーポイント（Anchor Point）、スケール、回転、および不透明度の初期値をJSON化する。

制約条件の手動設定: UI上で各部位の「可動限界値（例：膝は逆方向に曲がらない）」や「推奨加速度（Easing）」をユーザーが定義できる機能。

F-2: Supabase統合管理
要素別JSON保存: キャラクター全体を1つのJSONにするのではなく、「骨格」「外見（スタイル）」「アニメーションプリセット（歩行、挨拶等）」の各要素を個別のレコードとしてSupabase DBに保存する。

バージョン管理: 設定変更ごとにスナップショットを保存し、過去の構成（プロンプトの出し分け）を呼び戻せるようにする。

検索・フィルタリング: 登録済みのキャラクターや部位パーツをメタデータ（タグ、作成日）で検索する。

F-3: AIオーケストレーション
動的コンテキスト注入: 選択されたキャラクターのJSONをSupabaseから取得し、あらかじめ用意された「AE専門家プロンプト」と結合してLLMに送信する。

JSXコード生成・最適化: AIが生成したコードを、AEのES3環境で動作するように自動で型変換・エラー回避コード（try-catch）を付与する。

2. 非機能要件定義書 (Non-Functional Requirements Specification)
拡張機能の安定性とスケーラビリティを担保するため、以下の基準を厳格に適用する。

2.1 パフォーマンス・効率性
起動速度: パネル起動からUI操作可能になるまで2.0秒以内。

非同期実行の徹底: * AIとの通信、Supabaseへの保存、AEへのインジェクションはすべて別スレッド（Node.jsのworkerプロセスの活用を検討）で管理し、AEのGUIをフリーズさせない。

JSX実行効率: evalScript に渡す文字列は、冗長なコメントを削除し、最小限のペイロードに圧縮してインジェクションする。

2.2 セキュリティ・堅牢性
Supabase RLS (Row Level Security): * キャラクターデータはユーザーIDに紐付け、auth.uid() = user_id による厳格なアクセス制御を実施。

シークレットキーの隠蔽:

クライアント（AE）側にAPIキーを保持させない。Supabase Edge Functionsを経由し、サーバーサイドでAPIを叩く。

コードインジェクション対策:

AIの回答からコードブロック（```javascript）のみを抽出するパーサーを実装。

OSコマンド（system.callSystem, File.remove）をJSX内で実行させないための文字列バリデーションをCEP側で実施。

2.3 信頼性・可用性
データ同期の整合性: * Optimistic UI（楽観的UI）を採用。Supabaseへの保存完了を待たずにUI側は完了表示を出し、バックグラウンドでリトライ処理を行う。

エラーハンドリング: * ネットワーク遮断時に「オフラインモード」に自動切り替え。ローカルのIndexedDBに一時保存し、オンライン復帰時にSupabaseとコンフリクト解消（Last Write Wins）を行う。

2.4 保守性・拡張性
スキーマ駆動開発: * JSONデータ構造をTypeScriptのInterfaceとSupabaseの型定義（Generated Types）で完全に一致させる。

プラグインアーキテクチャ: * 将来的に他のDCCツール（Blender、Premiere）へ移植しやすいよう、AIロジック部分をAE依存のコードから分離したモジュール（Pure JS）として構築する。

3. システムアーキテクチャ図（概念）
UI (Next.js): キャラクターの「型」を定義し、Supabaseと通信。

Logic (Node.js/CEP): AEから情報を抜き出し、AIの回答をJSXとしてインジェクション。

Data (Supabase): キャラクターの構造（JSON）を要素ごとにストック。
